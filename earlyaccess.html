LITENAI RELEASE NOTES

INTRODUCTION

LitenAI is an AI engineer automating technical support and debug.

LitenAI is a chatbot that orchestrates customized AI agents based on user prompt to observe, reason and visualize petabyte scales of data. Liten lake ingests structured data, logs or metrics and enables accelerated search through big data. It ingests customer knowledge base from text, pdf, html or other media files. This knowledge base is used by agents to provide unique insight from the securely integrated knowledge and structured data lake. LitenAI left shifts technical support,and reduces response times to critical issues by 100x by removing engineering expertise bottlenecks.

It is avaiable for early access now. You can access it on public clouds with prepopulated agentic flow and lake. You can evaluate it by uploading your data there. It is also available as a docker image to try it out on your machine under your secure data networks. Please contact us to try out early access program or for any other information.

DOCKER INSTALL DIRECTIONS

Please download the docker image from LitenAI store. Once downloaded follow the following directions to run it locally. All data is local and accessible only to the user. LitenAI does not collect any data from this use.

  • Ensure that docker is installed and daemon running. USER should also be in the docker group. Follow the directions from docker website
  • Load the image from the downloaded image.For this load the image as shown below. Click here for more details.
    docker load < litenai.v0.0.78.tar.gz
  • The image should now be in docker images. This should show LitenAI image and tag.
    docker image ls
    Pick the litenai tag version that you want to run.
LitenAI agents can be used for technical support or debug from application logs or from equipment logs from the field. We show the two use cases below.

STARTING LITENAI SERVICE

To run the docker to analyze application logs set OpenAI API Key variable. Set API key to a valid OpenAI API key if using OpenAI. Set it to the API key if a local install is being used. It should be OpenAI API compatible.
export LITENAI_API_KEY="api_key"
If being served locally, the url to be used for LLM call and LLM Model being used are needed. LitenAI supports many LLMs. We also have local LLM installation images, we can help with deploying and serving LLMs of interest. Set these only if local LLM serve is being used. An example setting is shown below.
export LITENAI_SERVE_URL="http://localhost:8000/v1"
export LITENAI_LLM_MODEL="meta-llama/Llama-3.2-1B-Instruct"
LitenAI agents can be tuned to customer requirements. By default, the docker file provides a local lake with AI agents tuned for applicaiton log reasoning. For log reasoning you dont need to set these variables. However, if you wish to try out the mode for debug assistance for field equipments such as medical devices, LitenAI provides an agentic onfiguration with sample lake tuned to assist field technicians providing technical support for medical and industrial device. If you wish to use in this techassist mode. Please set these variables to enable this mode.
LITENAI_AGENTIC_MODE='techassist'
LITENAI_LAKE_URL='/srv/lake/techassist'
These agentic modes are data driven configurations and can be tuned to customer specifications.
Now you can run the docker command. Replace the version with the correct tag. Use only those variables that you set up for the agentic mode that you are trying out.
docker run -d --rm --name litenai_container -p 8080:8080 -p 8210:8210 -p 8220:8220 -e LITENAI_API_KEY=${LITENAI_API_KEY} -e LITENAI_SERVE_URL=${LITENAI_SERVE_URL} -e LITENAI_LLM_MODEL=${LITENAI_LLM_MODEL} -e LITENAI_AGENTIC_MODE=${LITENAI_AGENTIC_MODE} -e LITENAI_LAKE_URL=${LITENAI_LAKE_URL} litenai/litenai:version
It takes a few minutes for LitenAI service to start. Please see below for some sample chats. In the chatbot, in the side panel, you would also see saved chats. You can double click on them to load and see. You can chat with LitenAI as you would with a support and debug engineer. From the browser, you can open this location and start using the LitenAI.
http://localhost:8210
It will ask for login credentials. Please contact us to get the login credentials.

INGEST UNSTRUCTURED DATA

To add unstructured data like pdf, text, html, or any other media files, pick the tab named 'Lake' from main display. Pick the file under the heading.
Ingest unstructured data like pdf, text, html etc.
Click on ingest. It ingests all the data and indexes with embeddings to enable semantic search as well RAG (retreival augmented generation) flow.
You can add all your knowledge including pdf manuals, trouble shooting guidelines, standard operating procedures etc. LitenAI agents tune their response using this data. Note that none of this is shared with Liten or LLM, it all sits within customer lake.

INGEST STRUCTURED DATA

To add structured data from files containing structured logs or other tables, first pick the tab named 'Lake' from main display. Look at the following section.
Ingest csv or json. Infer new structures from file, and create table below.
All ingested structured data get appended to selected table.
If a table with that structure, already exists, pick the table from top. Choose the file. Supported file types are csv or json. Pick the options and then Ingest. It would add the data to the tables. 'Chat' tab shows the response.
If a table with file structure does not exist, it should first be created. Pick the table from top, it would show a sample json structure in the section titled.
Table Structure
To infer this from the selected file, enter a unique table name below this, and click on InferDesc. It shows the table structure json inferred above. Sometimes, the timestamp columns are not inferred properly. You may want to add this manually if timestamp is of importance. The fields are following
"timeseries": "timestamp", "timeseries_format": "yyyy-MM-dd HH:mm:ss.SSS"
Click on create button.
If successful, it would add it to be Table list and select it as an active table.
Now you can ingest the file by clicking on ingest button below the Ingest csv line.
Now you have added your structured logs to lake. This log can be reasoned our using LitenAI agents described above. Please go ahead and try out your reasoning scenarios.
